Reference
=========

Exhaustive, look-up-oriented pages: every built-in component name, CLI flag,
environment variable, and HTTP endpoint. Come here when you already know what
you want to do and need the exact name, option, or default.

**Start here:** go straight to the :doc:`component catalogue <components/index>`
when you are writing YAML, :doc:`cli` when you are scripting commands, and
:doc:`environment` when you are debugging what the runtime wrappers set. The
remaining pages cover the app-server HTTP contract and the Python API.
