# KonfAI v1.6.0

**Previous release:** v1.5.9 (2026-07-06). **Delta:** 15 commits on `main` (+34 957 / −631 over 217 files),
plus the `fix/review-findings` review branch (37 commits) and the v1.6.0 release-audit fixes.

v1.6.0 makes KonfAI's model layer **declarative** (a YAML model catalog + a registry-safe builder), bridges
external pretrained checkpoints into KonfAI graphs by execution order, ships a new **agent-facing MCP
server** (`konfai-mcp`), auto-detects OME-Zarr / DICOM stores, adds an **image-registration app
orchestrator**, and lands verified GPU-accumulation inference gains — without breaking the lazy patch-based
core or existing configs.

> Versions are derived from the git tag (`setuptools_scm`) across `konfai`, `konfai-apps`, and `konfai-mcp`.

---

## Major capabilities

### Declarative YAML model catalog + safe builder
- 14 shipped catalog models selectable by name with `classpath: default|<Name>.yml` — `UNet`, `NestedUNet`,
  `UNetPlusPlus`, `AttentionUNet`, `DynUNet`, `PlainConvUNet`, `ResidualEncoderUNet`, `SegResNet`, `UNETR`,
  `ViT`, `VNet`, `ResNet`, `ResNet18`, `VGG16` — or point at any `.yml` of your own.
- The builder (`konfai/utils/model_builder.py`) is **safe by construction**: node types come from two curated
  registries, module names contain no `.`, and there is no `eval`/import injection. Verified in this
  release's audit against 10 arbitrary-code-execution payloads (`!!python/*` tags, `$object`, `__globals__`
  traversal, dotted types) — all rejected with `ConfigError`.
- New parametric building blocks (`ResidualBlockD`, `ResNetBasicBlock`, `ResidualStage`, `DecoderStage`,
  `UNetPlusPlusNode`, `MultiHeadSelfAttention`, `PositionalEmbedding`, `ClipNormalize`), verified
  byte-identical to their torchvision / MONAI / nnU-Net references.

### Pretrained-weight bridge
- `konfai.utils.pretrained.transfer_weights_by_execution_order` loads an external checkpoint (MONAI,
  torchvision) into a KonfAI catalog graph by pairing weighted leaves in **forward-execution order** — no
  hand-written key map. Real torchvision ResNet-18 ImageNet weights reproduce the reference logits
  byte-for-byte (max abs diff 0.0).

### `konfai-mcp` — agent experimentation server (new package)
- A FastMCP server so an LLM agent can inspect a dataset → author YAML → run train / predict / evaluate →
  monitor jobs → compare runs → iterate.
- Safety, verified by execution: jobs run in a `spawn` subprocess; config validation and component
  smoke-tests run **only** in a spawn subprocess (never in the server process); `cancel_job` reaps the whole
  process group (`setsid` + `killpg`, so DDP grandchildren die with it); session file I/O is path-jailed.
- Published to PyPI on release — and, as of this release, **gated on its own test suite** in the publish
  pipeline.

### Store-format auto-detection & imaging
- Dataset readers auto-detect an **OME-Zarr / DICOM** store from its on-disk markers; DICOM series that carry
  no file extension are now recognised.
- Patch reads stay strictly lazy: reading one 0.52 MB patch from a **524 MB** volume adds **~0 MB** of RSS
  (measured).

### Apps & Slicer
- `impact_reg` registration app orchestrator, with declared-default optional inputs (`app.json "default"`),
  plus app-CLI smoke tests.
- App bundles: `impact_synth`, `impact_seg`, `impact_reg`, `mrsegmentator`, `totalsegmentator`.
- SlicerKonfAI contract preserved — every symbol Slicer imports from `konfai_apps.app_repository`
  (including `current_free_vram`) exists with a compatible signature.

### Verified performance
- **GPU accumulation** for patch reassembly: overlap-blended reassembly runs on-device and is
  **byte-identical** to the CPU path across Mean / Cosinus / Gaussian / no-blend (pinned by a new
  CUDA-gated test). Peak VRAM stays bounded to roughly the result + weight buffers.
- `IMPACTReg` gains a torch PCA reduction matched to `itk-impact`'s `pca_fit` / `pca_transform`, fit **per
  sample**.
- `SitkFile` statistics now stream instead of materialising and float64-casting the whole volume.

## Bug fixes (highlights)

- **The pretrained bridge no longer silently skips weights.** A module that owns parameters *and* has a child
  (`torch.nn.MultiheadAttention` holds `in_proj_weight` beside its `out_proj` child) was never treated as a
  leaf, so its weights stayed at random init while the transfer reported success. The bridge now raises if
  any **target** tensor is unreached; unused **source** branches (nnU-Net deep-supervision heads) are still
  correctly ignored.
- **VoxelMorph registers in 2-D and 3-D** (and rotates); a `shape` that contradicts `dim` is rejected.
- **NestedUNet no longer accepts an `attention` it does not implement**; **ResNet blocks** get the shape
  their residual add expects; **MIND** keeps a constant out of its denominator; the YAML catalog honours the
  parameters it exposes.
- **Config binding**: a `None` default for an optional nested object is honoured; a callable whose module
  writes annotations as text binds correctly; a non-string `Literal` default recovers its typed member.
- **MCP**: job records survive a crash; the runner survives its child; user code stays out of the server
  process and volumes out of server RAM; eight app/experiment tool contracts corrected.
- **Apps**: the server counts what it actually writes, and where; symlinks are walked once; bare DICOM is
  detected; bundle subpackages ship.
- **Data**: every rank's epoch is the same length; a rank holding nothing still gets an epoch to walk; a
  foreign random draw gives the process back its RNG state.

## Security changes

- **konfai-apps requirements guard canonicalizes names (PEP 503)** on both sides, so a protected core package
  (`torch`, `konfai`, `konfai-apps`) cannot be bypassed by spelling it `konfai_apps` or `Torch`. The
  docstring now states honestly that *transitive* dependencies are not policed.
- **MCP `prepare_dataset_aliases` rejects path-traversal group names** — previously an unsanitized
  `target_group` could copy/move dataset bytes anywhere the server user could write.
- **MCP config validation restores the authored config even when its subprocess times out** (the child's
  `finally` is skipped by SIGTERM; the parent now snapshots and restores).
- **The `default|<Name>.yml` catalog resolver is confined to the shipped catalog directory** (path
  separators rejected).
- **Config writes are atomic** (`os.replace` via a sibling temp file), so a concurrent reader can never
  observe a truncated/empty config and silently bind all defaults.

## Breaking changes & migration

- **Built-in model package moved:** `konfai/models/<kind>` → `konfai/models/python/<kind>`.
  - **Unaffected:** relative classpaths (`segmentation.UNet.UNet`), `default|<Name>.yml`, `module:Class`,
    and every shipped example, doc, and released HF bundle.
  - **Affected:** a config naming a built-in model by its **absolute** internal path
    (`konfai.models.<kind>.<file>:<Class>`). It still resolves, now emitting a `DeprecationWarning`.
    **Migrate to** `konfai.models.python.<kind>.<file>:<Class>`; the shim will be removed in a later release.
- **VoxelMorph**: `shape` must agree with `dim` (the 3-D default `shape=[192,192,192]` is valid and
  unchanged). A contradictory pair now raises instead of building a broken model.
- No config-file format change. Reading a config still resolves and rewrites it in place — now atomically.

## Known limitations

- The `konfai-mcp` segmentation E2E uses a wall-clock first-metric deadline that can flake under heavy CPU
  load; run it on an unloaded runner or raise the timeout.
- Multi-GPU / multi-node (SLURM) and the SlicerKonfAI GUI are validated by their own CI and manual runs, not
  by the core unit suite.
- TotalSegmentator MR / 3 mm variant bundles: ensure the referenced `.pt` weights are present in the
  published HF repo before use.
- The `Attribute` geometry sidecar round-trips `Origin`/`Spacing`/`Direction` through an 8-significant-digit
  string form — adequate for imaging geometry, but not bit-exact.

## Install & upgrade

```bash
pip install -U konfai                 # core
pip install -U konfai[imaging]        # + SimpleITK / h5py / pydicom / zarr / ngff-zarr
pip install -U konfai-apps            # app runner + FastAPI server
pip install -U konfai-mcp             # agent MCP server
# other extras: itk, hdf5, dicom, omezarr, tensorboard, lpips, ssim, fid, smp, export, cluster
```

The v1.6.0 wheel ships the full `konfai/models/python/**` tree and the `konfai/models/yaml/*.yml` catalog
(verified in a clean, non-editable install); `apps/` and `konfai-apps/` are excluded from the core wheel.
